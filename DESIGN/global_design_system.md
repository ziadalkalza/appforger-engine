# Global Design System

This is AppForge's reusable design foundation.

## Rule

Project-specific design rules can add to this file. Approved Figma baselines override visual implementation rules for the active project.

## Default qualities

- Clean.
- High-fidelity.
- Minimal when possible.
- Mobile-native.
- Strong hierarchy.
- Clear touch targets.
- Consistent spacing.
- Accessible contrast.
- Reusable components.
- Platform-aware behavior.

## Design tokens to define per project

- Primary color.
- Secondary color.
- Background colors.
- Text colors.
- Success/warning/error.
- Font scale.
- Border radius.
- Elevation/shadow.
- Spacing scale.
- Icon style.
- Illustration style.
- Motion style.

## Component defaults

- Buttons.
- Cards.
- Inputs.
- Navigation bars.
- Tabs.
- Sheets/modals.
- Lists.
- Empty states.
- Error states.
- Loading states.


## Imported design systems

The global design system is the default fallback only. Stronger design taste can be injected per project through `DESIGN_SYSTEM_IMPORTS/`.

Imported systems must be registered, license-checked, mapped to native mobile, and approved before use.

Do not edit this file for every project. Use `DESIGN/project_design_override_rules.md` and `DESIGN_SYSTEM_IMPORTS/PROJECT_DESIGN_STACK.md` for project-specific design layers.
