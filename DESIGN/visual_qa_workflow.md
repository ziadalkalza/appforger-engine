# Visual QA Workflow

## Purpose

Ensure native implementation matches approved Figma.

## Checks

- Layout.
- Spacing.
- Typography.
- Colors/tokens.
- Components.
- Images/assets.
- Motion.
- Empty/error/loading states.
- Platform-specific behavior.
- Accessibility.

## Process

1. Capture app screenshots.
2. Compare to Figma baseline.
3. Classify mismatches.
4. Patch implementation or open design change request.
5. Update test registry.
