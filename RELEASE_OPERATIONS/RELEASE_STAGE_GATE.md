# Release Stage Gate

## Required before entering release candidate

- target product/platform versions selected
- release scope frozen
- critical features implemented
- critical tests defined
- known issues listed
- documentation readiness checked

## Required before final release approval

- required tests passed or formally waived
- release blockers cleared or high-risk override recorded
- production environment validated
- store/web metadata prepared
- rollback/hotfix plan exists
- user/project owner approval captured
