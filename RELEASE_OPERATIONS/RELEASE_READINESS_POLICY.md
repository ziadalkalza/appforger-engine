# Release Readiness Policy

A release may proceed only when relevant evidence exists:

- release candidate created
- platform release scope defined
- required tests passed or waived according to policy
- release blockers resolved or explicitly overridden
- production environment checks completed
- required documentation updated
- release notes drafted and approved
- screenshots/store/web assets prepared if relevant
- final release approval recorded

A task, feature, or app may be marked ready only based on evidence, not verbal confirmation alone.
