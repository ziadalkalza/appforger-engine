# Task Closeout Checklist

Before closing a task, confirm:

- [ ] Task ID exists in task registry.
- [ ] Task packet exists or equivalent instructions are recorded.
- [ ] Source-of-truth baselines are listed.
- [ ] Expected output exists.
- [ ] Acceptance criteria are checked.
- [ ] Tests/builds are recorded or explicitly waived.
- [ ] Changed files are listed.
- [ ] Commit/branch/PR are recorded if available.
- [ ] A done report is written.
- [ ] Affected registries are updated.
- [ ] Follow-up tasks are created for warnings/missing evidence.
- [ ] Required approval is recorded or marked pending.

If any item is missing, close only as:

- `completed_with_warnings`,
- `user_marked_done_needs_evidence`, or
- `manually_closed`.
