# Task Reopen Policy

A closed task can be reopened when:

- QA fails,
- user rejects the output,
- design/API baseline changes,
- dependency changes,
- hidden bug appears,
- completion evidence was wrong or incomplete.

## Reopen Flow

1. Record reason for reopening.
2. Link original task and evidence.
3. Identify affected artifacts.
4. Create follow-up task or change request.
5. Preserve previous completion record.
6. Update status to `reopened` or create successor task.

Do not delete old task history.
