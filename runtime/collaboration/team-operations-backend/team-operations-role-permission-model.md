# Role Permission Model

Default role examples:
- Appforger Operator: manage workflow, packets, sync, audits.
- Product Owner: approve scope, stories, release scope.
- Designer: approve design/visual baselines.
- Backend Lead: approve API/data model/backend readiness.
- Frontend Lead: approve frontend readiness.
- QA Lead: approve test evidence and release QA status.
- Admin/DevOps: manage production environment and secret metadata.

One person may hold multiple roles. In single-user mode, the user may hold all roles.
