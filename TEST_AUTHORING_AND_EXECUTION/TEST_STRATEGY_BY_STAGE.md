# Test Strategy by Stage

## Requirements stage
Create draft product tests from acceptance criteria and risk areas.

## Design stage
Add screen-state tests, visual QA checks, interaction tests, and accessibility expectations.

## Backend stage
Add endpoint tests, schema/migration checks, permissions, auth, and integration checks.

## Frontend stage
Add component/viewmodel/state/API-client tests, visual QA screenshots, and manual device/browser checks.

## Integration stage
Add end-to-end and cross-repo contract tests.

## Release stage
Run critical regression, privacy/security, accessibility, performance, and deployment/store-readiness checks.
