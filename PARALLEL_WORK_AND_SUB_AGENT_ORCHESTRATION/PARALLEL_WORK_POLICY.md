# Parallel Work Policy

Parallel work is allowed only with a formal parallel work plan.

The plan must answer:
- what agents are active
- what each agent owns
- what each agent depends on
- which repos/branches/paths each agent may touch
- which baselines are fixed, draft, or changing
- which conflicts are possible
- which outputs must be reconciled
- what blocks completion

Do not parallelize when coordination cost is higher than the benefit. Prefer sequential work when requirements are unclear, architecture is unsettled, the same files will be touched, or the same baseline is actively changing.
