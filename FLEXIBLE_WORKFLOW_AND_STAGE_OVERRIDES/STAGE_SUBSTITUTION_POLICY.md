# Stage Substitution Policy

A substitution means the project still needs an output, but it comes from a different route.

Examples:
- Figma design stage substituted by approved HTML sandbox visual baseline;
- Figma stage substituted by imported existing frontend screenshots and component inventory;
- backend design substituted by imported existing backend API documentation;
- manual QA substituted by approved automated test evidence only when coverage is sufficient.

A substitute must declare what canonical artifact it replaces and what review is required before promotion.
