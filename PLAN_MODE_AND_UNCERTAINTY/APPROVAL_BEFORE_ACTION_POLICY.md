# Approval Before Action Policy

Require explicit approval before:

- activating a new integration,
- using paid services,
- creating production cloud resources,
- changing backend stack,
- changing architecture pattern,
- approving design/API baselines,
- running destructive migrations,
- pushing/merging code,
- storing or using production secrets,
- release/deployment actions.

Approval can be recorded as:

```text
Approved by user: yes
Date:
Scope:
Limitations:
```
