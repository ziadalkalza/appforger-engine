# Question Batching Policy

Ask only useful questions. Batch related questions to reduce back-and-forth and token waste.

## Good pattern

Ask 3-7 grouped questions when they affect architecture or integrations.

## Bad pattern

Ask one tiny question per turn or ask for information already present in project-control.

## If many details are missing

Use plan mode and separate questions into:

- required now,
- can decide later,
- safe default.
