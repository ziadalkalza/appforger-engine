# Real Work Start Gate

Before real implementation begins, verify:

- app mode selected,
- workspace/project-control created,
- target repo exists or remote route exists,
- source-of-truth baseline/spec exists or task is explicitly exploratory/mock-only,
- unsupported integrations have approved packs,
- required connector/MCP/skill statuses are active or fallback is selected,
- required approvals are recorded,
- task packet exists for Codex work.

If any item is missing, AppForge should enter plan mode or bootstrap/validation mode, not pretend the system is ready.
