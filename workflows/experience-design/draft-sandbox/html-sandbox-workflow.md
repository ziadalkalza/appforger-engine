# HTML Sandbox Workflow

HTML sandboxes are allowed for both mobile and web projects as quick visual/interaction experiments.

## Purpose

Use HTML sandboxing to test:

- layout direction;
- interaction sequence;
- copy density;
- simple animation behavior;
- visual hierarchy;
- component arrangement.

## Rules

- HTML sandbox is not production frontend.
- HTML sandbox does not replace SwiftUI, Compose, or approved web frontend code.
- Do not copy sandbox code directly into production without a real implementation packet.
- If useful, promote the result into design notes, Figma prompt, screen spec, or execution packet.

## Output

Store raw files in:

```text
local-only/drafts/html-prototypes/<draft-id>/
```

Store screenshots/review notes in:

```text
exports/drafts/<draft-id>/
```
