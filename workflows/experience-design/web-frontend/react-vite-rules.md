# React + Vite Rules

Use React + Vite + TypeScript for lightweight SPAs, dashboards, prototypes, or apps with a separate backend.

## Rules

- Use TypeScript.
- Centralize routing, API clients, state, and design tokens.
- Keep feature components modular.
- Do not hardcode API URLs.
- Use mock data only when a task packet explicitly marks the task as mock-only.
- Implement loading, empty, error, and permission states for real screens.
