# Engine Ready-for-Use Checklist

Appforger is ready for a project when:

- engine folder exists and is not mixed with app code,
- project-control is created from templates,
- app mode is selected or pending explicitly,
- required repos are local or remotely accessible,
- connector/MCP/skill status is tracked,
- third-party integrations are either supported or routed through upgrade workflow,
- plan mode and uncertainty rules are active,
- external workspace validation passes,
- task completion/evidence rules are active,
- context/token rules are followed,
- user knows how to start and continue sessions.
