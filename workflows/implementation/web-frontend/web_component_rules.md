# Web Component Rules

Components must map back to approved Figma components or documented project design system primitives.

## Required states

- default
- hover
- focus-visible
- active/pressed
- disabled
- loading where applicable
- error where applicable

## Rules

- Components should be accessible by default.
- Interactive components must support keyboard navigation.
- Use semantic HTML where possible.
- Do not implement one-off visual components if a reusable component exists.
