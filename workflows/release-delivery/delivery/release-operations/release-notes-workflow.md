# Release Notes Workflow

Release notes are generated in plan mode first.

Sources:
- user stories marked done
- completed feature registry entries
- changelog entries
- QA reports
- known issues
- release blocker outcomes

Process:
1. Gather candidate changes.
2. Group into user-facing changes, fixes, internal changes, known issues.
3. Ask user to approve tone and level of detail.
4. Create release notes draft.
5. Record approval in release evidence registry.
