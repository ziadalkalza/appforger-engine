# User Stories and Product Backlog

This layer turns product/business input into epics, user stories, acceptance criteria, test cases, design/backend/frontend implications, and Codex work packets.

User stories are required for product/user-facing features. Technical/internal tasks may use technical task specs instead.

Story creation should start in plan mode. Appforger should propose the backlog, ask for approval, then write approved/draft stories to external `project-control/`.
