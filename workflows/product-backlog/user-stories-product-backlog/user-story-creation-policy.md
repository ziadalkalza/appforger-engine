# User Story Creation Policy

## Default format

Use both forms:

```text
As a [actor], I want [goal], so that [benefit].
```

and structured fields:

```text
Story ID:
Epic:
Actor:
Goal:
Reason:
Status:
Acceptance criteria:
Dependencies:
Related screens:
Related APIs/data:
Related tests:
Related execution packets:
Documentation impact:
```

## Approval

In single-user mode, the user/project owner approves all user stories.

## Generation timing

After product brief, Appforger enters plan mode and proposes epics/stories. It must ask for approval before generating a large backlog.
