# Project-Control Collaboration Policy

In team mode, `project-control/main` should be protected.

All project-control changes should use PRs.

Low-risk PRs may use fast review. High-risk PRs require named approvers.

High-risk examples:
- baseline approval/change
- API contract change
- release candidate approval
- integration activation
- package/library decision
- security/privacy change
- context backend indexing scope change

Low-risk examples:
- meeting note added
- typo correction
- task progress note
- non-canonical draft note
