# Re-onboarding Policy

Re-onboarding is needed when project fundamentals change.

Examples:
- Single-user mode changes to team mode.
- Backendless project adds backend requirements.
- HTML sandbox workflow switches to Figma-first.
- Provider/tool-consistency mode changes.
- Context backend moves from local JSONL to Qdrant/Supabase.
- Automation level changes from manual to agent execution.

Re-onboarding should ask only affected sections, not the full questionnaire unless requested.
