# Package Selection Policy

Before introducing a package/library, Appforger must:

1. Identify the exact need.
2. Check whether platform-native or existing project code is enough.
3. Research current package options using official/package-registry sources when current facts matter.
4. Compare license, maintenance, security, popularity, compatibility, bundle/app size impact, and complexity.
5. Ask the user before adding the dependency.
6. Record the decision in external `project-control/REGISTRIES/PACKAGE_DECISION_REGISTRY.md`.

If research is unavailable, the recommendation is draft-only.
