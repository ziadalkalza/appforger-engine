# HTML Sandbox to Visual Baseline

HTML sandbox work may be used to explore high-fidelity UI for web or mobile-inspired interfaces.

Flow:
1. create draft/sandbox record;
2. build local-only or draft-branch HTML prototype;
3. iterate until user is satisfied;
4. run review checklist;
5. extract screens, components, states, interactions, data needs, and responsive behavior;
6. create visual baseline candidate;
7. user approves;
8. update screen, visual baseline, design asset, and backend implication registries;
9. generate backend/API and implementation packets.

The sandbox code itself is not production frontend unless explicitly imported and approved as such.
