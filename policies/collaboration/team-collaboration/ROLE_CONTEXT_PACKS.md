# Role Context Packs

A role context pack is a small, focused set of files/sections that a person should load for their role. It reduces tokens and prevents accidental edits outside their responsibility.

## Backend Developer pack
Read: project state, relevant user stories, API registry, data model registry, backend packets, security/env rules, active change requests.

## Frontend Developer pack
Read: project state, relevant user stories, screen registry, design baseline, API baseline, frontend packet, test cases.

## Designer pack
Read: product brief, relevant user stories, screen registry, design assets, Figma roundtrip rules, design baseline registry.

## QA pack
Read: acceptance criteria, test case registry, completion registry, baselines, open bugs, release blockers.

## Release pack
Read: release registry, evidence registry, blockers, documentation readiness, deployment/store checklists.
