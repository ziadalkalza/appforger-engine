# No-Backend App Policy

Some projects do not need a backend.

Examples:
- static marketing site;
- frontend-only prototype;
- offline-only utility;
- local demo;
- pure content site.

Before skipping backend, AppForge must ask whether the app needs auth, forms, payments, database, storage, admin panel, realtime, analytics, or server-side data. If not, backend can be marked skipped_not_required.

If needs change later, backend stage may be reopened through a change request.
