# Package Research Protocol

Research should prioritize:

1. Official package documentation.
2. Official registry metadata such as npm, PyPI, Swift Package Index, Maven/Gradle, CocoaPods, GitHub releases.
3. Security advisories and known vulnerabilities where relevant.
4. License files.
5. Maintenance signals: recent releases, issues, compatibility.

Do not speculate about package health, licensing, pricing, API shape, or platform compatibility.
