# Team Conflict Resolution

Conflicts must be recorded, triaged, and resolved openly.

Conflict statuses:
- detected
- triage_required
- under_review
- resolution_selected
- resolved
- deferred
- accepted_risk

Resolution locations:
- Git/file conflict → GitHub PR/rebase/merge process
- Design conflict → Figma + design baseline change request
- API conflict → API review + API baseline update
- Product conflict → Product Owner decision
- validation/domains/qa/release conflict → release blocker review
- Task-board conflict → project-control wins unless explicitly changed
- Context backend conflict → reindex/freshness validation
