# Capability Activation Checklist

Before a connector, MCP, skill, or integration is used in a real task, confirm:

- capability exists in engine docs or integration pack,
- status is not merely `requested` or `generated`,
- required user approvals are recorded,
- secrets/config are stored in correct locations,
- read/write permissions are known,
- validation check passed or explicit fallback chosen,
- affected registries are updated,
- task packet references the active capability.
