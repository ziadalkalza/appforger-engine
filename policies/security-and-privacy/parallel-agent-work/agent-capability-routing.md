# Agent Capability Routing

Sub-agents are routed by capability first, then provider.

Capabilities include:
- product planning
- UX flow planning
- design/prototype
- code editing
- repo inspection
- test execution
- QA review
- release preparation
- context retrieval/indexing
- security/privacy review

Use the provider layer to select a suitable provider. If the current tool lacks capability, create an execution packet for the proper provider or continue draft-only.
