# Integration Decision Packet Policy

Every integration should produce an integration decision packet.

Default path:

`project-control/integrations/<integration-id>/integration-decision-packet.md`

Required sections:

- Integration name and goal
- User problem to solve
- Candidate approaches considered
- Connector/MCP research summary
- Existing AppForger integration availability
- Selected approach
- Why this approach was selected
- Security assessment
- Cost/hosting assessment
- Source-of-truth and storage decision
- RAG/vector decision
- Graph decision
- Credential placement
- Setup steps
- Commands
- Approval requirements
- Testing/iteration notes
- Open risks
