# Docs Folder Structure

Recommended `docs/` structure:

```text
docs/
  README.md
  meeting-notes/
  product/
  design/
  backend/
  frontend/
  qa/
  release/
  legal-privacy/
  research/
  external/
  imports/
  summaries/
  archive/
```

Use `docs/imports/` for original imported files and `docs/summaries/` for Markdown summaries/extractions used by the context backend.
