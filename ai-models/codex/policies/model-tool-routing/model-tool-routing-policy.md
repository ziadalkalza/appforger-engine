# Model and Tool Routing Policy

Before high-impact work, Appforger must recommend the best environment.

If the current environment is not ideal, Appforger must offer two paths:

1. Proper tool path — create the correct packet/handoff for Codex, Figma, image generation, or third-party app.
2. Draft-only current-tool path — continue here with clear limitations and mark output as untested/unapplied/unapproved.

The user may continue in the current tool after the warning, but Appforger must record that limitation when relevant.
