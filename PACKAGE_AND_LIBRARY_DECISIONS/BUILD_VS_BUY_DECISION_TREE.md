# Build vs Buy Decision Tree

Prefer native/platform code when:
- the feature is small,
- the dependency would be heavier than the implementation,
- security/licensing is unclear,
- platform APIs already solve the problem.

Prefer an external package when:
- the domain is complex,
- security/compliance requires battle-tested providers,
- the package is actively maintained,
- the package reduces risk/time significantly,
- the license is compatible.

Always ask user before adding.
