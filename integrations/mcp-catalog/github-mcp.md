# GitHub MCP

## Role

Use GitHub MCP for:

- Reading repos.
- Creating branches.
- Reviewing files.
- Creating issues.
- Creating pull requests.
- Tracking implementation tasks.

## Rules

- Do not commit directly to main.
- Prefer branch + PR.
- PR descriptions must reference Appforger artifacts.
- Do not edit unrelated areas.
- Do not include secrets.

## Required project state

- Repo URL.
- Branch policy.
- Which repo role: backend, iOS, Android, engine docs, design assets.

## Handoff

Update `REGISTRIES/REPO_REGISTRY.md` and relevant task files.
