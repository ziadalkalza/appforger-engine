# Image Generation Usage Policy

Use image generation for visual exploration and asset creation when the project does not already have approved assets.

## Default uses

- app logo directions
- app icon explorations
- mascot concepts
- onboarding illustrations
- empty states
- subscription/paid-feature visuals
- icon families
- store screenshot background art
- marketing concept art

## Required approval

Every generated image must be reviewed before it becomes a project asset.

Review against:
- brand fit
- visual consistency
- platform suitability
- copyright/trademark risk
- clarity at small sizes
- export requirements
- Figma import readiness

## Source of truth

Approved visual assets should be stored in:
- Figma file, and/or
- `design-assets/` repo

Appforger stores metadata and approval state, not the final asset source of truth.
