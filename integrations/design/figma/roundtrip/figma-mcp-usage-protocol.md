# Figma MCP Usage Protocol

Use Figma MCP when available to fetch structured design context for implementation.

## Allowed

- read approved frames/components
- inspect design variables/tokens
- extract component hierarchy
- write prototype/design assistance if user requested
- summarize design for native implementation

## Not allowed without approval

- replacing approved frames
- changing design baseline
- deleting components
- exporting unapproved assets as production assets

If MCP is unavailable, use manual Figma exports and screenshots.
