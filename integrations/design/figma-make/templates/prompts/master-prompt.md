# Figma Make Master Prompt

Use after UX flows and visual direction exist.

```text
Build a high-fidelity app prototype for the following Appforger project.

Product:
[insert product summary]

Target users:
[insert users]

App mode:
[generic spec-first / iOS first / Android first / parallel native]

Screens:
[insert screen list]

User flows:
[insert flows]

Visual direction:
[insert approved visual direction or options]

Design rules:
- Use the project-specific design system if provided.
- Otherwise use the Appforger global design direction.
- Create clean, implementation-ready components.
- Avoid clutter.
- Include loading, empty, error, and success states.
- Include reusable components and tokens.
- Include animation/motion notes where useful.
- Make it suitable for native SwiftUI and Kotlin Compose implementation later.

Output needed:
- Screen frames.
- Prototype links/interactions.
- Component names.
- Asset list.
- Animation notes.
- Any assumptions made.
```
