# Promote Draft to Project

Promotion is the process of turning draft work into project-controlled work.

## Promotion flow

```text
review draft
→ classify artifact type
→ check required evidence
→ ask user approval
→ move/copy output to correct location
→ update project-control registries
→ create/change baseline or execution packet if needed
```

## Promotion targets

| Draft type | Promotion target |
|---|---|
| HTML prototype | design notes, Figma prompt, screen spec, or web task packet |
| Figma draft | candidate/approved design baseline |
| image draft | design-assets/pending or approved asset registry |
| Codex draft code | real Codex task packet or draft branch promotion PR |
| mock API | API draft/contract proposal |
| package experiment | package decision record |
