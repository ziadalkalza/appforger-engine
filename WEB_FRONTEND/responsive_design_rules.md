# Responsive Design Rules

Web design baselines must define responsive behavior before implementation.

## Required breakpoints unless waived

- mobile
- tablet
- desktop
- wide desktop when relevant

## Required states

- responsive navigation
- content wrapping/truncation
- table/card behavior
- modal/drawer behavior
- overflow handling
- empty/loading/error states

Codex must not invent responsive behavior unless the task packet explicitly allows a proposal.
