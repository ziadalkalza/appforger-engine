# Next.js Rules

Use Next.js for production web apps that need routing, SEO, authenticated pages, server-side functionality, metadata, API route proximity, or strong deployment support.

## Rules

- Use TypeScript.
- Keep design tokens centralized.
- Separate UI components, feature modules, API clients, and server-only logic.
- Never expose secrets to the browser.
- Prefer server-side validation for sensitive actions.
- Use route groups and clear page structure.
- Use approved Figma responsive baselines before implementation.
- Do not generate pages without matching route, state, API, and QA entries.

## Required before Codex implementation

- Approved web stack
- Approved responsive Figma baseline
- API contract or explicit mock-only instruction
- Web task packet
- Environment strategy
