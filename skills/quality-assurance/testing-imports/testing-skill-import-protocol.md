# Testing Skill Import Protocol

1. Identify the external testing skill/source.
2. Capture source URL or uploaded file reference.
3. Check license and attribution requirements.
4. Classify testing scope:
   - backend
   - API
   - iOS
   - Android
   - web
   - visual QA
   - accessibility
   - security
   - performance
   - release QA
5. Extract reusable rules and templates.
6. Map rules to Appforger stages and task packets.
7. Record conflicts with existing Appforger testing rules.
8. Ask the user to approve activation.
9. Register activated skill in the testing skill registry.

Do not activate an external skill when license, source, or applicability is unclear.
