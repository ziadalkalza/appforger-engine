# Write Automated Tests Skill

## Purpose
Guide Codex to write automated tests in the correct implementation repo.

## Rules
- Test code belongs in the implementation repo, not Appforger engine.
- Use the repo's selected stack and test commands.
- Do not add heavy frameworks without approval.
- Update test evidence and done reports after running tests.

## Steps
1. Identify target repo and files.
2. Read relevant test cases and baselines.
3. Write minimal useful tests.
4. Run available test command.
5. Record results and missing evidence.
