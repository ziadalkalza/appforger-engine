# Requirements

This folder stores project-specific requirements truth.

The Business Requirements Document (BRD) is mandatory for real app-building work. If the user provides a BRD template, store the project-specific copy here and record it in `registries/requirements-document-registry.md`. If the user does not provide a template, start from `business-requirements-document.md`, which is copied from the reusable engine template.

The BRD sits above the product brief. The product brief, user stories, UX flows, design work, backend/API contracts, frontend implementation packets, and QA plans must reference the current approved or active draft BRD unless the project records an explicit waiver.
