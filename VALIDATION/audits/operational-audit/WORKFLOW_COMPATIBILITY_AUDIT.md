# Workflow Compatibility Audit

Checks whether all AppForge workflows can coexist.

## Workflows covered

- standard mobile native workflow;
- standard web workflow;
- generic/spec-first workflow;
- draft/sandbox workflow;
- flexible stage override workflow;
- existing frontend/backend import workflow;
- backendless/static workflow;
- team collaboration workflow;
- context-backend-assisted workflow;
- automation workflow;
- release candidate workflow.

## Compatibility principle

The recommended path remains useful, but it is not the only valid path. A project may skip, substitute, import, or reorder stages if it records the override, performs impact analysis, respects non-negotiable rules, and updates project-control.
