# End-to-End Scenario Tests

The audit should simulate these scenarios without building production apps:

1. Standard mobile app.
2. Standard web app.
3. HTML sandbox-first web app.
4. Backendless static web app.
5. Existing frontend import.
6. Existing backend import.
7. Team mode with project-control PRs.
8. Context backend with Qdrant.
9. Supabase context backend profile.
10. Automation level 2 validation/reporting.
11. Release candidate flow.
12. Package/library decision flow.
13. Trello task-board export flow.
14. Draft/sandbox promotion flow.
15. Tool/model routing warning flow.

Each scenario checks required files, registries, packets, approvals, non-negotiable rules, and expected warnings.
