# AppForge Clone Strategy

You may use AppForge in one of three ways:

## Copy per app

Copy the engine into each app workspace. Simple and self-contained.

## Shared engine repo

Keep AppForge as its own Git repo and clone it into each app workspace.

## Engine plus app-state repo

Keep reusable AppForge files in one repo and app-specific state in a separate repo. This is cleaner for multiple apps but requires more discipline.

For early use, copy per app is easiest.
